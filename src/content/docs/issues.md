# Issues

Issues are used to track bugs, enhancements, or general discussions that require team attention.

## Issues vs Tasks
While **Tasks** are planned work items, **Issues** are often unplanned or reactive items (like a bug report from a user).

## Lifecycle of an Issue
1. **Reported**: An issue is created with a description and severity.
2. **Triaged**: A manager reviews and assigns the issue.
3. **Linked**: The issue can be converted into a task or linked to a pull request.
4. **Resolved**: The fix is deployed and the issue is closed.
