# Manage Repositories

The **Repositories** manager is Wekraft's central dashboard for linking and organizing codebase repositories connected to your projects. Linking a repository enables real-time commit activity trackers, automated branch mapping, and file structure trees.

---

## Workspace Integration Flow

To link a code repository to a project:
1. Ensure your personal GitHub account is connected in your [App Sidebar Settings](/web/docs/app-sidebar#2-github-sync-hub).
2. Open the **Repositories** page from your global App Sidebar.
3. Browse the list of repositories from your GitHub account.
4. Click **Connect** next to any unlinked repository and choose the target Wekraft project.

### Connection Flow

1. **Connect GitHub Account** (in global Settings)
2. **Browse Repositories** (on Repositories Page)
3. **Select Unlinked Project** (from dropdown list)
4. **Confirm Linkage**
5. **Enable Heatmaps & Codebase Links** (auto-synchronized)


---

## Key Features

### 1. Linked Repository Popover
Click the **View Connected Repo** button to open a status popover showcasing all active connections:
- **Project Name**: Displays the title of the linked Wekraft project.
- **Repository Name**: Displays the GitHub path (e.g. `owner/repo-name`).
- **Workspace Navigation**: Includes a quick icon shortcut to open the project's dashboard.

### 2. Search & Filter
- Includes an inline search box to filter your connected repositories list by name, owner, or status.

### 3. Show Repo Component
- Displays a grid of repositories loaded from your GitHub account.
- Clearly flags repositories that are already synced with project workspaces, showing an **Active Connection** status badge.
- Lists unlinked projects in a dropdown to select and sync them on the fly.
